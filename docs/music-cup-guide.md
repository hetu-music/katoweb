# 河图音乐杯使用说明

## 启动项目

```bash
pnpm install
pnpm dev
```

打开：

- `http://localhost:3000/zh-CN/music-cup`
- 或 `http://localhost:3000/zh-TW/music-cup`

生产构建：

```bash
pnpm build
pnpm start
```

## 完成一场比赛

1. 在首页选择入池方式：河图初选、全库抽签，或登录后的我的收藏。全库/收藏超过 48 首会随机抽取，不足 48 首会自动随机补齐，并显示来源/抽样/补入数量。
2. 查看当前 48 首候选；全库/收藏模式可再次点击当前模式重抽，然后点击“开始抽签”或“确认入池并开始”。
3. 查看十二支抽签结果，点击“入子支”。
4. 每支择两曲，再点击“落印，赴下一支”。
5. 12 组结束后，从 24 首遗珠中选择 8 首。
6. 点击“八席既定，开三十二强”。系统会将 24 首直通与 8 首归席歌曲重新随机配对。
7. 在每场一对一对决中择一曲；落印动画结束后自动进入下一场。
8. 完成 31 场淘汰赛后查看魁首、次席与四席结果。

## 全库偏好排序

点击音乐杯首页底部的“进入全库偏好排序”，或直接打开 `/zh-CN/music-cup/preferences`：

1. 每轮查看两首随机但尽量有信息量的歌曲。
2. 选择 A 胜、B 胜或平局；系统更新两首歌的 rating、比较次数与不确定度。
3. 右侧层级列表会按当前 rating 聚合，底部有向图用箭头展示胜负关系。
4. 页面给出建议比较预算，但达到预算后仍可继续比较。少量比较得到的是近似排序，不代表永久或绝对的音乐价值。
5. 未登录时记录保存在浏览器；登录后页面会请求账号级 API，并同步登录后的新选择。既有访客历史不会自动合并到账号。要启用账号同步，先按 `docs/supabase-deployment.md` 应用偏好排序 migration。

## 返回、恢复与重置

- “返回上一步”撤销最近一次已经落印的结果。
- 小组或复活赛中尚未锁定的勾选会被自动保存；刷新后仍在。
- 淘汰赛每场均有独立历史快照；结果页返回会回到未选择胜者的决赛。
- “重开此局”会弹出确认框。选择“续入此局”不改变数据，选择“重开此局”清空当前进度。
- 结果页“再开一局”同样先确认，确认后回到候选首页。
- 本地保存 key 为 `hetu-music-cup:v1`。schema 或候选发生不兼容变化时，页面会安全丢弃旧数据。

## 分享结果

1. 在问鼎结果页点击“绘制晋级图”。
2. 等待“正在绘制问鼎之路…”完成。
3. 弹窗会显示一张包含魁首摘要和完整问鼎路径的竖版 PNG。
4. 点击“收下此图”下载。

封面会依次尝试封面 CDN、`hetu-music.com` 的 OG 封面代理和站内默认封面；全部加载失败时，分享图才使用文字色块继续生成，不会因此丢失晋级路径。

## 曲库与无后端模式

页面会优先读取项目曲库，用真实专辑、封面、歌手和音频状态增强候选数据。

如果本地没有有效 Supabase Service Role 配置，或曲库请求失败：

- 页面自动使用仓库内置的 48 首候选；
- 抽签、小组赛、复活赛、淘汰赛、保存、回退和分享仍完整可用；
- 内置条目保留实际歌曲 ID；独立封面缺失的条目会使用 `hetu-music.com` 页面提供的默认 OG 封面。

## 试听说明

音乐杯没有新增公开试听接口。播放按钮沿用网站现有权限：

- 已登录；
- 用户拥有试听权益；
- 歌曲标记为存在音频；
- 用户的 Navidrome 凭据和服务端配置有效。

不满足条件时播放按钮隐藏，但“曲目档案”始终可打开。比赛选择本身不依赖试听权限。

## 开发检查

```bash
pnpm exec tsx --test src/components/music-cup/music-cup-utils.test.ts
pnpm format:check
pnpm lint
pnpm exec tsc --noEmit
pnpm build:next
```

状态机测试会验证：48 首唯一、12 组各 4 首、24 首直通、8 首复活、32 强无重复、31 场淘汰赛、唯一冠军和旧存储拒绝。
