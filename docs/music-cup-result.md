# 河图音乐杯实现结果

## 最终入口

- 简体中文：`/zh-CN/music-cup`
- 繁体中文：`/zh-TW/music-cup`
- 本地开发示例：`http://localhost:3000/zh-CN/music-cup`

## 已实现功能

- 固定 48 首河图代表作候选，并在首页完整公开候选清单与“非热度榜”说明。
- 每场赛事重新随机抽签为子、丑、寅、卯、辰、巳、午、未、申、酉、戌、亥十二支，每支 4 首。
- 12 轮小组赛，每组选择 2 首，得到 24 首直通歌曲。
- 24 首遗珠中选择 8 首复活，和直通歌曲重新随机组成 32 强。
- 三十二强、十六强、八强、四席争魁、终局问鼎共 31 场一对一淘汰。
- 淘汰赛胜者落印后自动进入下一场。
- 魁首结果页展示魁首、次席和另外两首四席歌曲。
- 每个已提交步骤均可“返回上一步”，包括从冠军页撤销决赛。
- 重置确认、结果页再来一场、刷新恢复和损坏存储安全降级。
- 受权限约束的站内试听与始终可用的曲目档案入口。
- `1800 × 2800` PNG 分享长图，包含完整 32 强至冠军晋级树。
- 桌面、平板和手机响应式布局，并尊重“减少动态效果”系统设置。
- 曲库不可用时使用内置候选数据，不阻断赛事。

浏览器完整流程的本次确定性选择结果为：冠军“阳关调”、亚军“东流”，另外两首四强歌曲为“海棠酒满”和“又惊春”。该结果仅用于验证结果推导和分享图，不代表固定比赛结果；新赛事会重新随机抽签。

浏览器开发模式左下角显示的 3 个 CSP issue 来自现有项目禁止 `eval()`、而 React 开发调试代码需要 `eval()` 的差异；生产构建通过，与音乐杯业务代码无关。构建日志中的 Supabase `42501` 权限提示也是现有数据源权限问题，音乐杯已按设计回退到内置候选，不影响页面构建与赛事流程。

## 已知限制

- 候选清单是编辑代表作池，不是由播放量计算的榜单。
- 试听严格依赖现有登录、权益、歌曲音频标记和 Navidrome 配置；普通访客不会看到公开试听按钮。
- Local Storage 只在同一浏览器、同一站点中恢复，不做登录账号跨设备同步。
- 分享图显示文字 URL，不包含不可验证或无法维护的二维码。
- 用户清除浏览器站点数据后，未完成赛事无法恢复。
- 封面 CDN 返回 `403` 时，页面会自动改用 `hetu-music.com` 的 OG 封面代理；独立封面不存在时使用站内默认封面，代理也失败后才退化为青黛题字封面。
- 本次自动浏览器环境不能直接切换到手机视口，因此移动端仅完成 CSS 断点和横向溢出规则检查；建议发布前补一次 390px 和 430px 真机或 DevTools 交互回归。
