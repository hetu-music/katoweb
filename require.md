角色设定与目标
你是一位精通东方极简美学、微交互设计以及 React/Next.js 技术栈的资深前端工程师。请帮我重构一个具有高级感的“顺承式” Hero 区组件。

技术栈要求

动画：Framer Motion (用于实现阻尼感和丝滑的文本渐现)

设计语言与视觉规范

核心词：克制、文气、纯文字排版、留白。

绝对禁止：不要使用任何按钮（Button）样式、背景色块或生硬的边框。

交互逻辑（核心难点）
这部分包含两个平级的文本：
文本 A（主标题）：谣歌 （此处保持原样）
文本 B（副标题）：你一定想知道，戏里讲了什么故事……
文本 C（入口）：意象词云 ↗

需要实现“桌面端悬停渐现 + 移动端滚动渐现”的响应式双态逻辑：

状态一：桌面端（支持 Hover 的设备）

默认状态： 只显示文本 A B。文本 B 末尾附带一个“呼吸闪烁”的省略号…… （颜色极淡，透明度 40% 到 80% 循环）。隐藏文本 c。

交互状态： 当鼠标悬停在整个 Hero 容器区域时，闪烁光标透明度渐隐消失。文本 c 伴随轻微的水平位移（如从 translateX(-10px) 到 0）和透明度变化（opacity 0 到 1）平滑如水波般浮现。动画时长需较长（如 0.8s），带有 easeOut 缓动。

状态二：移动端（不支持 Hover 的触屏设备）

默认状态： 隐藏闪烁光标。只显示文本 A B。隐藏文本 c。

交互状态： 监听页面的垂直滚动（scrollY）。当页面向下滚动超过 20px 时，触发显示文本 c。文本 c 从文本 B 下方换行浮现，伴随极其轻微的垂直位移（translateY(10px) 到 0）和淡入效果。
